GitHub Integration Setup Guide

🔌 Connect GitHub Projects to This Dashboard

1
Generate a Personal Access Token in GitHub → Settings → Developer Settings → Tokens (Classic). Scopes needed: repo, read:org, project
2
Add token as a GitHub Actions secret: METRICS_TOKEN in your dashboard repo Settings → Secrets
3
Use the GitHub GraphQL API to query Projects v2 data: endpoint https://api.github.com/graphql — query your org's projectsV2 items and fields
4
Add a GitHub Actions workflow .github/workflows/metrics.yml to run the data fetch nightly and commit the results as data/metrics.json
5
This index.html reads data/metrics.json on load and renders real data. Enable GitHub Pages on the main branch root folder — your dashboard goes live at https://iPivotTechnology.github.io/dev-metrics
GitHub Actions Workflowmetrics.yml
name: Refresh Metrics
on:
  schedule:
    - cron: '0 0 * * *'
  workflow_dispatch:

jobs:
  fetch:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Fetch GitHub Metrics
        env:
          GH_TOKEN: ${{ secrets.METRICS_TOKEN }}
        run: node scripts/fetch-metrics.js
      - name: Commit data
        run: |
          git config user.name "metrics-bot"
          git add data/metrics.json
          git commit -m "chore: refresh metrics"
          git push
GraphQL Query ExampleProjects v2
query {
  organization(login: "iPivotTechnology") {
    projectsV2(first: 5) {
      nodes {
        title
        items(first: 100) {
          nodes {
            fieldValues(first: 10) {
              nodes {
                ... on ProjectV2ItemFieldTextValue {
                  text field { name }
                }
                ... on ProjectV2ItemFieldSingleSelectValue {
                  name field { name }
                }
              }
            }
            content {
              ... on Issue {
                title assignees { nodes { login } }
                closedAt createdAt labels { nodes { name } }
              }
            }
          }
        }
      }
    }
  }
}
